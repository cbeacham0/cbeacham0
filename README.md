> **Portfolio Note**
>
> The projects in this portfolio are intentionally designed to **simulate the real workflows of a Vulnerability Management and Security Engineering professional working in enterprise environments**.
>
> Each project focuses on practical security tasks such as asset discovery, authenticated vulnerability scanning, CVE analysis, exploitability validation, remediation tracking, rescanning, executive reporting, and compliance evidence.
>
> The goal is to demonstrate **how a cybersecurity engineer identifies, prioritizes, validates, and drives remediation of security risks at scale**, using tools such as **Nessus, Nmap, Kali Linux, Windows/Linux targets, Splunk, Power BI, Python, PowerShell, and NIST/RMF-style documentation**.

## Current Portfolio Goal

I am building this portfolio to demonstrate practical, job-ready cybersecurity experience in vulnerability management and security engineering.

The goal is to show the full lifecycle of vulnerability operations:

1. Discover assets
2. Identify exposed services
3. Run vulnerability scans
4. Validate findings
5. Prioritize risk
6. Create remediation plans
7. Track ownership
8. Rescan after remediation
9. Report risk to technical and executive stakeholders
10. Map evidence to security frameworks and compliance requirements
